# Angry Birds 3D

Фанатский 3D-ремейк знаменитой аркадной головоломки, основанной на физике. Разрабатывается на движке Unity для ПК. 

Цель проекта — перенести классическую механику стрельбы из рогатки и разрушения построек в трехмерное пространство, сохранив узнаваемый геймплей и улучшив визуальный опыт.

## Как поиграть

Игра находится в стадии активной разработки. Вы можете скачать последнюю тестовую сборку и попробовать ее уже сейчас.

### Установка:
1. Перейдите в раздел [Releases](../../releases) справа.
2. Скачайте последний архив с игрой `AngryBirds3D.zip`.
3. Распакуйте архив в любую удобную папку.
4. Запустите файл `Angry Birds 3D.exe`.

### Управление:
* **Запуск птицы:** Зажмите левую кнопку мыши на птице в рогатке, оттяните назад для прицеливания и отпустите.
* **Камера:** Камера автоматически следует за летящей птицей и переключается на рогатку при завершении хода. 
* **Оконный режим/полноэкранный режим:** `Alt + Enter`
* **Выход из игры:** `Alt + F4`

## Техническая информация

Проект разрабатывается с упором на чистую архитектуру, масштабируемость и использование современного стека технологий Unity. 

### Стек технологий:
* **Движок:** Unity 6
* **DI Контейнер:** [VContainer](https://vcontainer.hadashikick.jp/) — для внедрения зависимостей и построения Composition Root.
* **Асинхронность:** [UniTask](https://github.com/Cysharp/UniTask) — для работы с асинхронными операциями без аллокаций.
* **Реактивность:** [R3](https://github.com/Cysharp/R3) — для связывания данных и UI, а также обработки событий.
* **Анимации:** [LitMotion](https://github.com/AnnulusGames/LitMotion) — легковесная и быстрая библиотека для программных анимаций.
* **Камера:** Unity Cinemachine — для плавного переключения видов.
* **Система ввода:** Input System Package (New).
* **Ассеты:** Addressables.

### Архитектура и паттерны:
* **Паттерн Mediator:** Используется для слабой связности между классами и системами (например, `BirdQueuePigTrackerMediator`).
* **Осознанный нейминг:** В проекте действует строгое правило — никаких классов с именами `Manager` или `Controller`. Каждый класс имеет узкую зону ответственности (например, `SlingshotShooter`, `InputProvider`, `LevelStateTracker`).
* **Data Storage:** Синхронное сохранение/загрузка данных через `System.Text.Json` с использованием runtime-кэширования.
* **Object Pooling:** Используется для оптимизации создания часто используемых объектов (например, отображение очков и звуковые эффекты).
* **Code Style:** Использование лямбда-выражений для однострочных методов, приватные поля с нижнего подчеркивания, строгий контроль за жизненным циклом подписок (Disposable) для предотвращения утечек памяти.
